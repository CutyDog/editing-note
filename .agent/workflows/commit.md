# Git Commit ワークフロー

以下の手順でコミットを実行する。

## 1. ステージング状況の確認
// turbo
`git status --short` を実行して、現在の変更状況を確認する。

## 2. 差分の確認
// turbo
ステージ済みの変更がある場合は `git diff --cached --stat` で差分の概要を確認する。
ステージ済みの変更がない場合は `git diff --stat` で未ステージの差分を確認する。

## 3. 必要に応じてステージング
ステージ済みの変更がない場合、ユーザーに確認した上で `git add .` を実行する。
特定のファイルのみステージしたい場合はユーザーの指示に従う。

## 4. 詳細な差分の確認
// turbo
`git diff --cached` を実行して、コミット対象の詳細な差分を確認する。

## 5. コミットメッセージの生成
差分の内容に基づいて、以下のルールで日本語のコミットメッセージを生成する。

### コミットメッセージのルール
- **1行目（タイトル）**: 変更内容を簡潔に要約（50文字以内を目安）
  - プレフィックスを付ける: `feat:`, `fix:`, `refactor:`, `docs:`, `chore:`, `test:`, `ci:`, `style:`
  - プレフィックスの後は日本語で記述
- **3行目以降（本文）**: 変更の詳細を箇条書きで記述（必要な場合のみ）
  - 各項目は `- ` で始める
  - 何を変更したか、なぜ変更したかを簡潔に記述

### コミットメッセージの例
```
feat: ユーザー認証機能を追加

- JWTトークンによる認証を実装
- ログイン・ログアウトAPIを追加
- 認証ミドルウェアを追加
```

```
fix: ユーザー一覧のページネーションを修正

- オフセット計算の誤りを修正
- 総ページ数の算出ロジックを修正
```

```
chore: Docker環境のセットアップを追加
```

## 6. コミットの実行
生成したコミットメッセージをユーザーに提示し、問題なければ `git commit` を実行する。
ユーザーがメッセージの修正を求めた場合は修正してから実行する。
