# Lint 実行 & 修正ワークフロー

以下の手順でプロジェクト全体の静的解析を実行し、品質を維持します。

## 1. フロントエンドの Lint 実行
// turbo
`docker compose exec frontend npm run lint` を実行します。

- エラーが発生した場合は、出力内容から原因となっているファイルとルールを特定してください。
- 修正案を作成し、ユーザーに提示して「修正して良いか」を確認してください。

## 2. バックエンドの Lint 実行
// turbo
`docker compose exec backend bundle exec rubocop` を実行します。

- エラー（Offenses）が発生した場合は、対象ファイルと違反内容を確認してください。
- 修正案を作成し、ユーザーに提示して「修正して良いか」を確認してください。
- 自動修正（`--autocorrect`）が可能な場合でも、まずはどのような変更が行われるかを説明し、許可を得てください。

## 3. 修正の適用
ユーザーから許可（「OK」「やって」など）を得た場合のみ、ファイルの修正または自動修正コマンドを実行します。

## 4. 再確認
修正完了後、再度各環境の Lint コマンドを実行し、すべてのエラーが解消されたことを確認します。
